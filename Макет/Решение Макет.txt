Test plan 
Описание проекта 
•	Проект включает в себя функциональность фильтров и таблицу контактов. Фильтры располагаются на левом сайдбаре, а записи контактов отображаются в таблице справа. Пользователь может применять фильтры, изменять число отображаемых записей и переключаться между страницами таблицы. 
Цель тестирования 
•	Цель тестирования - проверить работоспособность функциональности фильтров, корректность отображения данных в таблице и соответствие интерфейса ожидаемому поведению. 
Критерии приемки 
1.	Все фильтры должны корректно применяться и влиять на отображение данных в таблице. 
2.	Таблица контактов должна отображать данные правильно и реагировать на изменения, вызванные фильтрами. 
3.	Интерфейс должен соответствовать макету и реагировать на действия пользователя согласно бизнес требованиям. 
Область проверки 
1.	Функциональность фильтров: 
•	Проверка работы фильтров по чек-боксам. 
•	Проверка применения фильтров после нажатия кнопки "Применить". 
•	Проверка корректной работы кнопок логики "И", "ИЛИ", "НЕ". 
•	Проверка сброса фильтров после нажатия кнопки "Сбросить". 
•	Проверка всплывающего уведомления после применения фильтров. • 	Проверка времени отображения всплывающего окна. 
2.	Таблица контактов: 
•	Проверка отображения данных в таблице. 
•	Проверка изменения числа записей в таблице после применения фильтров. 
•	Проверка работы бара переключения страниц. 
•	Проверка настройки отображения числа записей в таблице. 
•	Проверка всплывающего уведомления при изменении числа записей. 
3.	Интерфейс: 
•	Проверка отображения счетчика выводимого числа записей. 
•	Проверка корректного отображения кнопок "Применить" и "Сбросить". 
•	Проверка всплывающих уведомлений после действий пользователя. • 	Отсутствие ошибок и сбоев при выполнении всех действий. 
 
 
 
Test Cases  
1)Применение фильтра по языку (Russian): 
Шаги: 
1.	Выбрать чек-бокс "Russian". 
2.	Нажать кнопку "Применить". 
Ожидаемый результат: В таблице отображаются только записи с русским языком. Число записей соответствует указанному в фильтре. 
 
2)Применение фильтра по наличию имени (has first name): 
Шаги: 
1.	Выбрать чек-бокс "has first name". 
2.	Нажать кнопку "Применить". 
Ожидаемый результат: В таблице отображаются только записи, у которых заполнено поле "First Name". Число записей соответствует указанному в фильтре. 
 
3)Применение комбинации фильтров с логикой "ИЛИ": 
Шаги: 
1.	Выбрать чекбокс "Russian" и "English". 
2.	Убедиться, что логика "ИЛИ" активна. 
3.	Нажать кнопку "Применить". 
Ожидаемый результат: В таблице отображаются записи как с русским, так и с английским языком. Число записей соответствует сумме записей по этим фильтрам. 
 
4)Применение комбинации фильтров с логикой "И": 
Шаги: 
1.	Выбрать чекбоксы "Russian" и "has first name". 
2.	Переключить логику на "И". 
3.	Нажать кнопку "Применить". 
Ожидаемый результат: В таблице отображаются только те записи, которые соответствуют обоим фильтрам. Число записей корректно отображается. 
 
 
 
 
 
5)Применение фильтров с логикой "НЕ": 
Шаги: 
1.	Выбрать чекбокс "Russian". 
2.	Включить логику "НЕ". 
3.	Нажать кнопку "Применить". 
Ожидаемый результат: В таблице не отображаются записи с русским языком. Число записей корректно отображается. 
 
6)Сброс фильтров: 
Шаги: 
1.	Применить любой фильтр. 
2.	Нажать кнопку "Сбросить". 
Ожидаемый результат: Список записей возвращается в исходное состояние (10 000 записей). 
 
7)Проверка уведомлений: 
Шаги: 
1.	Выбрать любой фильтр (например "English" + "НЕ")  
2.	Применить фильтр. 
Ожидаемый результат: Появляется уведомление о применении фильтра, которое исчезает через 3 секунды. 
 
8)Переключение страниц : 
Шаги: 
1. Переключиться на вторую страницу таблицы. 
  Отображаются записи, соответствующие второй странице. 
 
 
9)Изменение числа отображаемых записей: 
Шаги: 
1. Изменить настройку вывода числа элементов (например на 50.) 
Ожидаемый результат: В таблице отображается 50 записей. 
 
 
 
 
 
11) (Негативный тест) Отсутствие результатов при применении фильтров: 
Шаги: 
1.	Выбрать фильтры, которые не пересекаются (например, "has account number" и "has telephone number" с логикой "И"). 
2.	Нажать кнопку "Применить". 
Ожидаемый результат: В таблице не отображаются записи, уведомление о нулевых результатах. 
 
Вывод : 
Данный тест-план описывает процесс функционального тестирования фильтров и таблицы контактов. После завершения тестирования ожидается подтверждение соответствия ожидаемым критериям приемки.
